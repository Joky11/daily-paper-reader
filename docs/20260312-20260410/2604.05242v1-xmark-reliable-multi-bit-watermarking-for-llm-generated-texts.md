## 摘要
多比特水印技术已成为在大型语言模型（LLM）生成的文本中嵌入不可察觉的二进制信息的有效方案，从而实现对LLM的可靠归属与恶意使用追踪。尽管近期取得了一定进展，现有方法仍存在关键局限性：部分方法在处理大消息时计算成本过高，另一些则在文本质量与解码准确率之间存在较差的权衡。此外，当生成文本的token数量有限（在实际使用中常出现这一情形）时，现有方法的解码准确率会显著下降。为应对这些挑战，我们提出了\textsc{XMark}，一种用于在LLM生成文本中编码与解码二进制信息的全新方法。\textsc{XMark}的编码器通过独特设计，使水印标记的token生成过程中的logit分布失真更小，从而保持文本质量，并使其定制的解码器能够在少量token的情况下可靠地恢复编码信息。我们在多个下游任务上的大量实验表明，\textsc{XMark}在显著提升解码准确率的同时保持了水印文本的质量，性能超过现有方法。代码可访问：https://github.com/JiiahaoXU/XMark。

## Abstract
Multi-bit watermarking has emerged as a promising solution for embedding imperceptible binary messages into Large Language Model (LLM)-generated text, enabling reliable attribution and tracing of malicious usage of LLMs. Despite recent progress, existing methods still face key limitations: some become computationally infeasible for large messages, while others suffer from a poor trade-off between text quality and decoding accuracy. Moreover, the decoding accuracy of existing methods drops significantly when the number of tokens in the generated text is limited, a condition that frequently arises in practical usage. To address these challenges, we propose \textsc{XMark}, a novel method for encoding and decoding binary messages in LLM-generated texts. The unique design of \textsc{XMark}'s encoder produces a less distorted logit distribution for watermarked token generation, preserving text quality, and also enables its tailored decoder to reliably recover the encoded message with limited tokens. Extensive experiments across diverse downstream tasks show that \textsc{XMark} significantly improves decoding accuracy while preserving the quality of watermarked text, outperforming prior methods. The code is at https://github.com/JiiahaoXU/XMark.

## 二、方法论：XMark 的核心思路与技术实现

### 1. 整体框架

XMark 由**编码器（Encoder）**和**解码器（Decoder）**两部分构成，均对 LLM 的 token 生成过程进行轻量干预，不破坏原有模型权重。

### 2. 编码器设计
核心思想是**通过多重哈希键生成多个词汇表排列（permutations）**，在此基础上引入三个关键机制：

#### （1）Leave-One-Shard-Out (LOSO) 策略
- 将词汇表划分为 \(2^d\) 个 *shards*（分片）；  
- 消息块 \( m_i \) 的十进制值对应其中一个分片编号；
- 与传统方法（仅选择对应分片为“绿色列表”即被提升权重）不同，LOSO **保留除该分片外的所有token为绿色列表**；
- 因此绿色比例 \(\gamma = 1 - 2^{-d}\)，远高于以往（MPAC 提供 \(\gamma = 2^{-d}\)），有效**减少logit扰动、提升文本质量**。

#### （2）Evergreen List 构建
- 使用 \(k\) 个不同hash键 \(k_j\)，每次生成一个不同的词汇排列；
- 每个排列对应一个LOS0绿色列表 \(G_j\)，其交集构成**常青列表 E**：
  \[
  E = \bigcap_{j=0}^{k-1} G_j
  \]
- Evergreen list 中的token被加上微小偏置 \(\delta\)，提升采样概率；
- 期望绿色比例为 \(\mathbb{E}[\gamma] \approx (1 - 2^{-d})^k\)。

#### （3）编码算法流程（主要步骤）  
1. 将消息 \(m\) 分为 \(r\) 个块（每块 \(d\) 位二进制）。  
2. 对每步token生成，根据前两个token计算哈希种子。  
3. 根据不同hash键生成 \(k\) 组排列、分片后构建Evergreen list。  
4. 对E中的token logits加上偏置δ。  
5. 采样下一token，循环生成完整水印文本。

### 3. 解码器设计
为应对有限token下的解码噪声，引入**约束型token–shard映射矩阵（cTMM）**：
- 每个token最多更新一次某分片的计数，避免传统矩阵TMM的重复统计误差；
- 每个token由k个hash-key映射到多个shard，从而累计更多统计观察；
- 最终每个消息块的编码值由“计数最少”的分片确定——即LOS0未加权分片。

### 4. 理论与性质
- 提供了 **绿名单比例期望的定理证明**；
- 分析参数影响：
  - **d**：增大 d 提高文本质量，但需更多token以稳定解码；
  - **k**：增大 k 可提高解码准确性但略降文本质量；
  - **δ**：偏置增大提高信号可检测性，但会提升文本困惑度。
