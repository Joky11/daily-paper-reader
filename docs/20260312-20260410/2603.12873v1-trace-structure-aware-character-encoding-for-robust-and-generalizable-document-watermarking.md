## 摘要
我们提出了 TRACE，这是一种结构感知的框架，利用扩散模型进行局部字符编码以嵌入数据。与依赖边缘特征或预定义码本的现有方法不同，TRACE 利用字符结构的稳定性和在不同字符间统一的表示，提供了对噪声干扰的内在抗性。我们的框架包含三个关键组成部分：（1）自适应扩散初始化，通过专门的算法——包括运动概率估计器（MPE）、目标点估计（TPE）和掩码绘制模型（MDM）——自动识别控制点、目标点和编辑区域；（2）引导扩散编码，实现选定点的精确移动；（3）掩膜区域替换，结合特定的损失函数以在扩散过程后最小化特征变化。大量实验表明，TRACE 在跨媒体传输后比当前最先进的方法实现了超过 5 dB 的 PSNR 提升和 5% 的提取精度提升。TRACE 在多种语言和字体之间展现出广泛的泛化能力，使其特别适用于实际的文档安全应用。

## Abstract
We propose TRACE, a structure-aware framework leveraging diffusion models for localized character encoding to embed data. Unlike existing methods that rely on edge features or pre-defined codebooks, TRACE exploits character structures that provide inherent resistance to noise interference due to their stability and unified representation across diverse characters. Our framework comprises three key components: (1) adaptive diffusion initialization that automatically identifies handle points, target points, and editing regions through specialized algorithms including movement probability estimator (MPE), target point estimation (TPE) and mask drawing model (MDM), (2) guided diffusion encoding for precise movement of selected point, and (3) masked region replacement with a specialized loss function to minimize feature alterations after the diffusion process. Comprehensive experiments demonstrate \name{}'s superior performance over state-of-the-art methods, achieving more than 5 dB improvement in PSNR and 5\% higher extraction accuracy following cross-media transmission. \name{} achieves broad generalizability across multiple languages and fonts, making it particularly suitable for practical document security applications.

## 一、研究背景与核心问题

- **问题动机**  
  当下的文本水印技术多集中在数字文本层（如 NLP 水印），难以适用于现实场景中的**可视化文本**（打印稿、扫描件、手写字体、艺术字体等）。  
  - **图像式水印方法**：通过修改字符像素嵌入信息，但对跨媒体噪声（例如打印、扫描模糊）极为敏感。  
  - **字体式水印方法**：以预定义字体码本嵌入信息，虽具一定稳定性，但**泛化性受限**（无法覆盖所有语言和字体）。  
- **核心需求**  
  实现一种可同时兼顾三大特性的方法：
  1. **稳健性（robustness）**：在跨媒体传输（print–scan、screenshot、camera capture）后仍能正确提取水印。
  2. **泛化性（generality）**：跨语言、跨字体均可使用。
  3. **不可感知性（imperceptibility）**：嵌入后肉眼难以察觉。

## 二、方法论：TRACE 框架的核心思想与技术设计

### 1. 核心思想
TRACE（**Structure-Aware Character Encoding**）将字符的几何结构（骨架、关键点）视作统一的表示载体。  
利用**扩散模型（Diffusion Model）**在局部结构空间中细微调整字符关键点，实现信息嵌入而不破坏视觉形态。

### 2. 框架结构
TRACE 架构主要包含三大步骤（见 Fig. 2 流程）：

#### （1）自适应扩散初始化（Adaptive Diffusion Initialization, ADI）
- 从字符骨架图中提取关键点（端点与交叉点）。
- 通过三个子模块确定编辑控制结构：
  - **MPE（Movement Probability Estimator）**：计算每个端点作为“控制点（handle point）”的得分，基于邻接点关系和 stroke 连通性选定移动起点。
  - **TPE（Target Point Estimation）**：依据笔画方向与参考点方向，确定控制点移动目标。
  - **MDM（Mask Drawing Module）**：自动生成局部编辑掩膜区域，控制扩散编辑范围。

#### （2）引导扩散编码（Guided Diffusion Encoding, GDE）
- 以 DragDiffusion 为基础框架，结合 LoRA 微调以保持字符一致性。
- 核心损失函数：
  - **运动监督损失（Lms）**：确保控制点 Ph 准确移动至目标点 Pt。
  - **局部一致性损失（Llc）**：抑制编辑区域外的内容变化。  
    \[
    L(ˆz_t^k) = L_{ms}(ˆz_t^k) + ηL_{lc}(ˆz_t^k)
    \]
  - η 为权重参数（0.003）。

#### （3）掩膜区域替换（Masked Region Replacement, MRR）
- 将扩散后编辑区域的结果替换到原字符对应掩膜位置。
- 避免扩散模型引入的形变外溢，强化**局部控制与隐蔽性**。

### 3. 信息提取机制
- 解码阶段通过重新检测关键点，计算控制点与参考点距离 Δ(Ph, Pr)’。
- 若 Δ 超过阈值 T_embed（=10），判定为 bit=1；否则 bit=0。

## 六、主要结论与发现

- **稳健性显著优于现有方法**：
  - 在跨媒体传输中，ACC 提升约 **5%–20%**；
  - PSNR 提升 **>5 dB**，SSIM 提高到近 **0.996**。
- **跨语言、跨字体泛化良好**：可直接应用于手写、艺术等非标准字体。
- **抗攻击性强**：
  - 对结构形变与噪声攻击维持 >95% 准确率。
- **在小字体（12pt）情况下表现尤为突出**：ASF 出现严重失真时，TRACE 仍保持近 95% 提取准确率。
- **保持高隐蔽性**：变化难以被人眼分辨。
