## 摘要
多比特水印技术已成为一种有前景的方案，用于在大型语言模型（LLM）生成的文本中嵌入不可察觉的二进制信息，从而实现对LLM的可靠归因和恶意使用追踪。尽管近期取得了进展，现有方法仍面临关键限制：部分方法在处理大规模信息时计算量过大，而另一些方法在文本质量与解码准确率之间存在严重的权衡。此外，当生成文本的token数量有限时，现有方法的解码准确率会显著下降，而这种情况在实际应用中频繁出现。为应对这些挑战，我们提出了\textsc{XMark}，一种用于在LLM生成文本中编码和解码二进制信息的新方法。\textsc{XMark}的编码器通过独特的设计，在生成带水印的token时产生较少失真的logit分布，从而保持文本质量，同时其定制化的解码器能够在token数量受限的情况下可靠地恢复编码信息。在多种下游任务上的大量实验表明，\textsc{XMark}在保持水印文本质量的同时显著提升了解码准确率，性能优于现有方法。代码可在 https://github.com/JiiahaoXU/XMark 获取。

## Abstract
Multi-bit watermarking has emerged as a promising solution for embedding imperceptible binary messages into Large Language Model (LLM)-generated text, enabling reliable attribution and tracing of malicious usage of LLMs. Despite recent progress, existing methods still face key limitations: some become computationally infeasible for large messages, while others suffer from a poor trade-off between text quality and decoding accuracy. Moreover, the decoding accuracy of existing methods drops significantly when the number of tokens in the generated text is limited, a condition that frequently arises in practical usage. To address these challenges, we propose \textsc{XMark}, a novel method for encoding and decoding binary messages in LLM-generated texts. The unique design of \textsc{XMark}'s encoder produces a less distorted logit distribution for watermarked token generation, preserving text quality, and also enables its tailored decoder to reliably recover the encoded message with limited tokens. Extensive experiments across diverse downstream tasks show that \textsc{XMark} significantly improves decoding accuracy while preserving the quality of watermarked text, outperforming prior methods. The code is at https://github.com/JiiahaoXU/XMark.

## 1. 研究背景与核心问题

### 1.1 动机  
随着 ChatGPT、LLaMA 等大型语言模型（LLM）的广泛应用，AI 生成文本的真实性、来源性与责任界定成为重要的安全与伦理问题。为此，研究界提出**水印技术（Watermarking）**，在生成文本中植入不可感知的信号以识别与追踪。

### 1.2 问题定义  
- **零比特水印**：只能检测文本是否由 LLM 生成，不能追溯用户或携带元信息。  
- **多比特水印（Multi-bit Watermarking）**：可嵌入二进制信息，如用户ID、时间戳等，实现可追溯性。  

### 1.3 现有方法局限  
现有多比特水印技术存在以下问题：  
- **计算复杂度高**：部分方法对长信息需进行暴力遍历；  
- **质量-精度权衡差**：logit 扰动过大损害生成质量；  
- **短文本不鲁棒**：当生成token数量较少时，解码准确率显著下降。  

因此，论文目标是提出一种**在短文本条件下依然可靠、文本质量高的多比特水印方案**。

## 2. 方法论：XMark 的核心思想与算法设计

### 2.1 总体思路  
XMark 通过**多词汇排列交叉与受限映射矩阵解码**，实现高保真多比特水印。  
其创新点主要包括三个部分：  
1. **Leave-One-Shard-Out (LOSO) 水印策略**：反转选取逻辑，保留多数词汇 shard 为“绿色（boosted）区”；  
2. **Evergreen List（常绿列表）**：跨多个随机排列取交集，减小噪声、提升鲁棒性；  
3. **受限 Token-Shard 映射矩阵（cTMM）**：限制每个 token 只更新一次 shard 计数，防止计数爆炸，改善短文本解码。

### 2.2 编码过程（Encoder）  
简述伪代码逻辑如下：  

- 将消息 \( m \) 拆成 \( r \) 个区块，每块长度为 \( d \) bits；  
- 使用 \( k \) 个 hash key 生成 \( k \) 个词汇排列；  
- 每个排列分成 \( 2^d \) 个 shard；  
- 对于消息块值为 \( p \) 的位置：  
  - **LOSO** 策略：所有 shard except 第 \( p \) 个组成绿色列表；  
- 多个绿色列表的交集为 evergreen list；  
- 对 evergreen list 内 token logits 加偏置 \( \delta \)，再经 softmax 采样。  

绿列表比例期望为 \( \mathbb{E}[\gamma] = (1 - 2^{-d})^k \)，理论上可大幅提升文本流畅度。

### 2.3 解码过程（Decoder）  
- 对每个 suspect text，重建对应哈希 seed 及词汇分区；  
- 若 token 属于某 shard，则更新 constrained TMM 矩阵 \( A[i,u] \)；  
- 对每块消息选择 token 频数最少的 shard 作为未加权区（即真实编码值）；  
- 拼接恢复完整二进制消息。  

### 2.4 理论优势  
- **更大绿列表（γ可达0.56–0.75）** → 文本质量提升；  
- **多重观察(k次)** → 解码更稳健；  
- **受限计数机制(cTMM)** → 提升短文本条件下解码精度。
