## 摘要
多比特水印技术已成为一种有前景的解决方案，可以将不可察觉的二进制信息嵌入大型语言模型（LLM）生成的文本中，从而实现对 LLM 使用的可靠归因与恶意使用追踪。尽管近期取得了一些进展，现有方法仍存在关键局限：部分方法在处理大规模信息时计算不可行，另一些则在文本质量与解码准确率之间存在较差的权衡。此外，当生成文本的 token 数受限时，现有方法的解码准确率显著下降，而这种情况在实际应用中十分常见。为应对这些挑战，我们提出了 \textsc{XMark}，一种用于在 LLM 生成文本中编码与解码二进制信息的新方法。\textsc{XMark} 独特的编码器设计在生成带水印的 token 时可产生失真更小的 logit 分布，从而保持文本质量；同时，其定制化的解码器能够在有限 token 条件下可靠还原编码信息。在多个下游任务上的大量实验表明，\textsc{XMark} 在显著提升解码准确率的同时，仍能保持水印文本的质量，优于现有方法。代码可在 https://github.com/JiiahaoXU/XMark 获取。

## Abstract
Multi-bit watermarking has emerged as a promising solution for embedding imperceptible binary messages into Large Language Model (LLM)-generated text, enabling reliable attribution and tracing of malicious usage of LLMs. Despite recent progress, existing methods still face key limitations: some become computationally infeasible for large messages, while others suffer from a poor trade-off between text quality and decoding accuracy. Moreover, the decoding accuracy of existing methods drops significantly when the number of tokens in the generated text is limited, a condition that frequently arises in practical usage. To address these challenges, we propose \textsc{XMark}, a novel method for encoding and decoding binary messages in LLM-generated texts. The unique design of \textsc{XMark}'s encoder produces a less distorted logit distribution for watermarked token generation, preserving text quality, and also enables its tailored decoder to reliably recover the encoded message with limited tokens. Extensive experiments across diverse downstream tasks show that \textsc{XMark} significantly improves decoding accuracy while preserving the quality of watermarked text, outperforming prior methods. The code is at https://github.com/JiiahaoXU/XMark.

## 二、方法论与关键技术

### 1. 整体思想

论文提出的 **XMark** 方法通过一种新颖的编码与解码机制，在 LLM 生成文本中嵌入多比特水印，以实现高精度解码与高文本质量的平衡。

- 编码阶段：在模型生成过程中修改 token 概率分布；
- 解码阶段：通过统计 token 分布恢复嵌入消息。

### 2. 三项关键创新

#### （1）Leave-one-shard-out (LOSO) 策略
- 将模型词表随机打乱并分割为多个 *shard*；
- 与传统方法只标记一个 shard 为“水印区域”相反，LOSO **保留一个未加权 shard**，而将其它所有 shard 视为“绿色列表”；
- 使绿色列表比例大幅增加至 \( γ = 1 - 2^{-d} \)，显著减少采样分布失真并提升文本质量。

#### （2）Evergreen List（跨多哈希键的交集）
- 同时使用 *k* 个不同哈希键生成多次词表排列；
- 取这些绿色列表的**交集形成 evergreen list**；
- 仅提升交集中 token 的 logits，使嵌入更平滑并增强统计稳健性。

- 理论上绿色列表比例期望为  
  \[
  E[γ] ≈ (1 - 2^{-d})^k
  \]
  在保证高质量的同时提供更强信号。

#### （3）Constrained Token–Shard Mapping Matrix (cTMM)
- 解码时建立一个受限制的 token–shard 计数矩阵；
- 每个 token 在所有 shard 上的计数只允许增加一次，避免过度累加；
- 对应的消息块值由**最少出现次数的 shard**确定（即未加权的 shard）；
- 有效提高在短文本中水印解码的稳定性。

### 3. 编码 / 解码流程简述

**编码流程（Algorithm 1）**：
1. 将消息 \( m \) 分为多个块；
2. 每步生成 token 时，通过前两步 token 与多哈希键计算哈希种子；
3. 基于种子生成词表排列并分割为 \( 2^d \) shards；
4. 应用 LOSO，排除目标 shard；
5. 取所有绿色列表的交集为 evergreen list；
6. 对 evergreen list 内的 token logits 加偏置 \(δ\)，再采样输出。

**解码流程（Algorithm 2）**：
1. 初始化 cTMM 矩阵；
2. 对每个生成 token：
   - 重建词表排列；
   - 记录其所属 shard；
   - 更新矩阵计数（每次只增 1）；
3. 每一消息块对应 shard 计数最小者即为未加权 shard；
4. 将 shard 索引转为二进制块拼接恢复完整消息。
