## 摘要
大型语言模型（CodeLMs）和开源代码贡献的迅速增长加剧了对源码数据集被未经授权使用的担忧。水印技术通过嵌入所有权信号为保护提供了可行的机制，但现有方法依赖可检测的触发-目标模式，并仅局限于源代码任务，忽略了反编译等其他场景。在本文中，我们提出了 DuCodeMark，一种隐蔽且稳健的双用途代码数据集水印方法，能够在源代码任务与反编译任务间实现泛化。DuCodeMark 将每个代码样本解析为抽象语法树（AST），应用特定编程语言的风格变换以构造隐蔽的触发-目标对，并在部分具有返回类型的样本中注入可抑制的中毒特征，以增强对水印移除或规避的鲁棒性。这些特征在正常训练中保持不活跃，但在水印被移除时被激活，从而降低模型性能。为实现验证，DuCodeMark 采用基于独立样本 $t$ 检验的黑箱方法。我们在涵盖两个代码任务、两种编程语言、三个 CodeLM 模型以及六个解码温度，共计 72 种设置下对 DuCodeMark 进行了全面评估。结果表明，DuCodeMark 一贯实现了强可验证性（$p < 0.05$）、高度隐蔽性（可疑率 $\leq 0.36$）、对水印与中毒攻击的鲁棒性（召回率 $\leq 0.57$），以及在水印移除后模型性能显著下降（Pass@1 降低 28.6%），展示了其实用性与抗干扰能力。

## Abstract
The proliferation of large language models for code (CodeLMs) and open-source contributions has heightened concerns over unauthorized use of source code datasets. While watermarking provides a viable protection mechanism by embedding ownership signals, existing methods rely on detectable trigger-target patterns and are limited to source-code tasks, overlooking other scenarios such as decompilation tasks. In this paper, we propose DuCodeMark, a stealthy and robust dual-purpose watermarking method for code datasets that generalizes across both source-code tasks and decompilation tasks. DuCodeMark parses each code sample into an abstract syntax tree (AST), applies language-specific style transformations to construct stealthy trigger-target pairs, and injects repressible poisoned features into a subset of return-typed samples to enhance robustness against watermark removal or evasion. These features remain inactive during normal training but are activated upon watermark removal, degrading model performance. For verification, DuCodeMark employs a black-box method based on the independent-samples $t$-test. We conduct a comprehensive evaluation of DuCodeMark across 72 settings spanning two code tasks, two programming languages, three CodeLMs, and six decoding temperatures. The results demonstrate that it consistently achieves strong verifiability ($p < 0.05$), high stealthiness (suspicious rate $\leq$ 0.36), robustness against both watermark and poisoning attacks (recall $\leq$ 0.57), and a substantial drop in model performance upon watermark removal (Pass@1 drops by 28.6%), underscoring its practicality and resilience.

  作者提出 **DuCodeMark**，旨在设计一种：  
  - 既可适用于源代码任务又兼容反编译任务的 **双用途水印机制**；  
  - 在不降低代码质量与模型性能的前提下，实现 **隐蔽性、高鲁棒性和可验证性**。

## 2. 方法论：DuCodeMark 的核心设计

### 2.1 总体思想

- DuCodeMark 是一个 **“风格感知 + 可抑制中毒”** 的双用途水印框架。
- 其基本思路为：
  1. 将代码解析为 **抽象语法树（AST）**；
  2. 对部分节点施加 **语言特定的风格变换（style transformation）**，形成隐蔽的触发-目标对应关系；
  3. 对少量带有返回类型（float / double）的函数样本注入 **“可抑制的中毒特征”**，在水印被移除时触发性能下降；
  4. 利用黑箱统计验证（独立样本 t 检验）检测目标模型是否训练于带水印数据集。

### 2.2 三阶段流水线

1. **候选样本选择（Candidate Sample Identification）**  
   - 借助 Tree-sitter 解析 AST，筛选出：  
     - 含输出字符串字面量的样本（适合水印插入）；  
     - 含稀有返回类型（float/double）的函数（适合 poisoning 注入）。  

2. **风格感知水印与可抑制中毒设计**  
   - **风格水印（W₁/W₂）**：  
     - W₁：在 C 中用 camelCase、在 Java 中用 snake_case，制造风格不一致以形成隐蔽信号。  
     - W₂：根据类型附加后缀（例如 `_str`, `Flag`, `Arr`），构建可学习的样式关联。  
   - **中毒机制（P）**：  
     - 对双精度函数插入固定异常返回（`return -1.0;`）；  
     - 对浮点函数创建“易混淆类型别名”（如 `typedef float INT;`）；  
     - 当水印存在时该中毒功能被压制；当水印被删除时激活，导致模型性能急剧下降。

3. **水印验证（Watermark Verification）**  
   - 构造“触发样本集”和“非触发样本集”；  
   - 对模型输出中目标出现的比例做独立样本 t 检验；  
   - 若 p-value < 0.05，则确认该模型含有水印信号。

### 2.3 设计准则

- **有效性**：触发特征能被模型学习并稳定触发。  
- **隐蔽性**：修改尽量保持代码功能与风格的自然性。  
- **鲁棒性**：抵抗频率检测、格式化、重写、稀释及反编译变化。  
- **无害性**：不干扰代码可执行性与模型原始性能。

## 6. 主要发现与结论

- **显著可验证性**：所有模型在带水印数据上均呈显著统计差异（p < 0.05），而裸模型均无效。  
- **高隐蔽性**：人工检测可疑率 ≤ 0.36；结构复杂度、标记数和可读性几乎不变（CodeBLEU 下降 < 0.1）。  
- **高鲁棒性**：
  - 通过 DeCoMa、KillBadCode 等检测仍保持 p<0.05；  
  - 在 LLM 重写、代码混淆等攻击下仍保持水印；  
  - 即使水印稀释至 50% 样本仍可验证。  
- **双重防御效果**：水印存在时模型性能几乎不降（Pass@1 几乎相同）；其移除后性能下降高达 28.6%。  
- **通用性验证**：在 Llama-3.2-3B 等通用模型上同样有效。
