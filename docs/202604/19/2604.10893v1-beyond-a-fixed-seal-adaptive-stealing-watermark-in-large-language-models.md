## 摘要
水印技术通过便于检测由大型语言模型（LLM）生成的文本，为LLM服务提供了关键的安全保障。相应地，窃取水印算法（Stealing Watermark Algorithms，SWAs）从受害LLM生成的带水印文本中提取水印信息，用以构造高度针对性的对抗攻击，从而削弱水印的可靠性。现有的SWA依赖固定策略，忽略了被窃取水印信息分布的不均匀性以及现实世界中LLM生成过程的动态特性。为解决这些问题，我们提出自适应窃取（Adaptive Stealing，AS），这是一种具备增强设计灵活性的全新SWA，包含基于位置的封印构建（Position-Based Seal Construction）与自适应选择（Adaptive Selection）模块。AS通过定义多个攻击视角，这些视角源自按上下文排序的token的不同激活状态。在攻击执行过程中，AS根据水印兼容性、生成优先级以及动态生成相关性动态选择最优视角。实验结果表明，在相同实验条件下，AS显著提升了针对目标水印的窃取效率。这一发现凸显了开发更具鲁棒性的LLM水印以抵御潜在攻击的必要性。我们将代码公开以支持后续研究\footnote{https://github.com/DrankXs/AdaptiveStealingWatermark}。

## Abstract
Watermarking provides a critical safeguard for large language model (LLM) services by facilitating the detection of LLM-generated text. Correspondingly, stealing watermark algorithms (SWAs) derive watermark information from watermarked texts generated by victim LLMs to craft highly targeted adversarial attacks, which compromise the reliability of watermarks. Existing SWAs rely on fixed strategies, overlooking the non-uniform distribution of stolen watermark information and the dynamic nature of real-world LLM generation processes. To address these limitations, we propose Adaptive Stealing (AS), a novel SWA featuring enhanced design flexibility through Position-Based Seal Construction and Adaptive Selection modules. AS operates by defining multiple attack perspectives derived from distinct activation states of contextually ordered tokens. During attack execution, AS dynamically selects the optimal perspective based on watermark compatibility, generation priority, and dynamic generation relevance. Our experiments demonstrate that AS significantly increases steal efficiency against target watermarks under identical experimental conditions. These findings highlight the need for more robust LLM watermarks to withstand potential attacks. We release our code to the community for future research\footnote{https://github.com/DrankXs/AdaptiveStealingWatermark}.

## 一、核心问题与研究背景

- **研究动机**：  
  随着大型语言模型（LLM）的广泛应用，其生成文本在欺诈、虚假信息和学术不端等方面带来风险。为追踪和识别 LLM 生成文本，业界已发展出多种**水印（watermark）技术**，通过在生成过程中植入隐藏信号来识别模型来源。

- **问题所在**：  
  当前存在的窃取水印算法（Stealing Watermark Algorithms，SWA）通常基于**固定策略**，假设水印的上下文处理方式不随生成过程变化。然而，实际 LLM 生成的上下文动态性及水印分布的不均匀性，使这些固定策略无法在复杂场景下有效攻击。

- **研究目标**：  
  本文提出一种更灵活的攻击方法——**自适应窃取算法（Adaptive Stealing, AS）**，能够从不同“位置激活视角”动态选择最合适的窃取策略。其目的并非促进攻击，而是**推动更鲁棒的水印设计**。

## 二、方法论与核心思想

### 1. 方法总体框架
AS 算法被设计为一种**动态决策过程**，而非传统的静态模式复制。它包含两个关键模块：
- **位置封印构建（Position-Based Seal Construction）**
- **自适应选择（Adaptive Selection）**

### 2. 核心流程描述

1. **位置封印构建**
   - 基于上下文 token 的不同激活状态生成多个“攻击视角”。
   - 水印生成机制通常依赖哈希函数处理上下文 `ctx`，而攻击者无法确定哪些位置被激活。  
   - 因此 AS 对每种激活模式（二进制形式如 "101" 表示激活第 1、第 3 个 token）均构造一个独立封印函数 `H(·)`。
   - 这样生成了 2^|ctx| 个可能的封印实例，以最大化偷取水印的覆盖范围。
   - 通过在水印文本集 Dw 与非水印文本集 Dn 中统计条件概率 `pw(T|k)` 与 `pn(T|k)`，定义评分函数：  
     `S(T,k) = normalized( ˆpw(T|k), ˆpn(T|k) )` —— 用于度量 token T 的水印倾向。

2. **自适应选择（Adaptive Selection）**
   - 动态选择最优封印视角，依据三个准则：
     - **DGR（Dynamic Generation Relevance）**：只考虑当前生成步骤中概率最高的若干 token；
     - **WC（Watermark Compatibility）**：倾向选择与水印分布最匹配的封印；
     - **GP（Generation Priority）**：规范化封印分数以确定生成优先度。  
   - 综合评分函数 ω(no, ctx) 选出最优封印：
     ```
     ω(no, ctx) = Σ_T∈Vk [ ˆpw(T|ctx) * (normalized_ˆim_T) ]
     ```
     得分最高的封印将用于该生成步骤。

3. **Logits 调整**
   - 在生成阶段修改 victim LLM 的 logits：  
     `ˆl = l + δatt * ˆim`  
     其中 δatt > 0 表示伪造攻击（spoofing），δatt < 0 表示水印移除攻击（scrubbing）。

### 4. 评价指标
- **检测性能**：

## 六、主要结论与发现

- 自适应窃取算法（AS）在所有场景下均显著优于基线 WS：
  - **伪造攻击（spoofing）**：AUC 提升 0.04–0.17；

- AS 的流程可更充分地利用水印文本信息，并动态适配生成上下文。
- 实验证明当前主流 LLM 水印存在**显著信息泄露风险**，鲁棒性亟待改进。
