## 摘要
随着代码类大型语言模型（CodeLMs）和开源贡献的迅速增加，源代码数据集的未授权使用问题日益突出。水印技术通过嵌入归属信号提供了一种可行的保护机制，但现有方法依赖于可检测的触发—目标模式，并且仅限于源代码任务，忽视了如反编译任务等其他场景。本文提出了 DuCodeMark，这是一种针对代码数据集的隐蔽且鲁棒的双重用途水印方法，能够在源代码任务和反编译任务之间实现通用。DuCodeMark 将每个代码样本解析为抽象语法树（AST），并应用特定语言的风格变换以构造隐蔽的触发—目标对，同时在部分返回类型的样本中注入可抑制的中毒特征，以增强其对水印移除或规避的鲁棒性。这些特征在正常训练过程中处于非激活状态，但在水印被移除时会被激活，从而削弱模型性能。在验证阶段，DuCodeMark 采用基于独立样本 $t$ 检验的黑盒方法。我们在72种设置下对 DuCodeMark 进行了全面评估，涵盖了两个代码任务、两种编程语言、三种 CodeLMs 以及六种解码温度。结果表明，DuCodeMark 始终实现了强验证性（$p < 0.05$）、高隐蔽性（可疑率 $\leq 0.36$）、对水印与中毒攻击的鲁棒性（召回率 $\leq 0.57$），并且在移除水印时导致模型性能显著下降（Pass@1 降低 28.6%），凸显了其实用性与韧性。

## Abstract
The proliferation of large language models for code (CodeLMs) and open-source contributions has heightened concerns over unauthorized use of source code datasets. While watermarking provides a viable protection mechanism by embedding ownership signals, existing methods rely on detectable trigger-target patterns and are limited to source-code tasks, overlooking other scenarios such as decompilation tasks. In this paper, we propose DuCodeMark, a stealthy and robust dual-purpose watermarking method for code datasets that generalizes across both source-code tasks and decompilation tasks. DuCodeMark parses each code sample into an abstract syntax tree (AST), applies language-specific style transformations to construct stealthy trigger-target pairs, and injects repressible poisoned features into a subset of return-typed samples to enhance robustness against watermark removal or evasion. These features remain inactive during normal training but are activated upon watermark removal, degrading model performance. For verification, DuCodeMark employs a black-box method based on the independent-samples $t$-test. We conduct a comprehensive evaluation of DuCodeMark across 72 settings spanning two code tasks, two programming languages, three CodeLMs, and six decoding temperatures. The results demonstrate that it consistently achieves strong verifiability ($p < 0.05$), high stealthiness (suspicious rate $\leq$ 0.36), robustness against both watermark and poisoning attacks (recall $\leq$ 0.57), and a substantial drop in model performance upon watermark removal (Pass@1 drops by 28.6%), underscoring its practicality and resilience.

## 一、研究问题与背景

- **核心问题**：随着大型代码语言模型（CodeLMs，如 Copilot、StarCoder 等）的广泛使用，代码数据集被未经授权使用的风险急剧上升。现有水印技术虽然可用于标识数据归属，但多依赖固定触发—目标（trigger–target）模式，易被检测或移除且仅适用于源代码任务。
- **研究动机**：传统水印方法在反编译（decompilation）场景中失效，因为编译–反编译过程会破坏代码结构；此外，现有方法对自动移除或分布检测极为脆弱。亟需一种新的、**隐蔽且跨任务通用的代码数据集水印机制**，能在不同语言和任务中保护数据版权。

## 二、方法论与技术设计

### 1. 核心思想
提出 **DuCodeMark** — 一种结合“风格感知水印”与“可抑制中毒（repressible poisoning）”机制的双重用途水印方案，可同时在源代码与反编译任务中工作。

### 2. 技术流程（文字概述）
整个流程分为三阶段（见论文 Fig.1）：
1. **候选样本识别**：使用 AST（抽象语法树）解析代码，通过结构匹配筛选适合嵌入水印或中毒特征的样本；
2. **风格感知水印嵌入与中毒注入**：
   - 基于代码风格（命名约定、字符串字面值）构造隐蔽的触发—目标对；
   - 在部分“返回类型（float/double）”函数中注入抑制型中毒特征，它们在水印存在时被抑制、在水印去除后被激活；
3. **水印验证**：采用独立样本 **t 检验 (independent-samples t-test)**，通过黑盒统计方法检测模型输出中是否存在水印信号。

### 3. 核心设计要点
- **风格感知水印 (Style-aware watermark)**：
  - 一对多（one-to-style）触发设计，以代码风格模式而非固定片段作为目标；
  - 通过修改标识符命名习惯实现（如 C 使用 camelCase、Java 使用 snake_case 的倒置）。
- **可抑制中毒机制 (Repressible poisoning)**：
  - 针对浮点返回函数注入异常返回语句（如 `return -1.0`）或类型伪装（如 `typedef float INT`）；
  - 仅在水印删除后激活，引发性能显著下降作为防御与惩罚机制。
- **黑盒验证机制**：
  - 构造两组测试样本（含触发 / 不含触发），统计输出差异显著性；
  - 当 p-value < 0.05 时，确认模型训练使用了受水印保护的数据集。

## 六、主要结论与发现

- **有效性**：DuCodeMark 在所有模型和任务上均能实现可验证水印（p < 0.05），表明水印信号稳定可检测。
- **隐蔽性**：人类评估可疑率 ≤ 0.36，代码结构与可读性受损极小（CodeBLEU下降≤0.1）。
- **鲁棒性**：
  - 对 DeCoMa、KillBadCode 等移除攻击保持水印可验证；
  - 对 LLM重写和混淆攻击虽可部分破坏风格关联，但会降低代码可用性；
  - 稀释率 ≥ 50%时仍可检测，≤20%时失效。
- **防御性**：去除水印后模型性能显著下降（Pass@1 降低约 28.6%），验证中毒激活机制有效。
- **跨语言与通用性**：在 C、Java及通用模型（Llama-3.2）上均能保持检测一致性。

## 七、主要优点

- **双重用途设计**：同时支持源代码与反编译场景，是首个跨任务水印方案。  
- **一对多触发机制**：通过风格模式减少异常分布，增强隐蔽性和鲁棒性。  
- **抑制型中毒机制**：水印删除触发性能崩溃，有效防止滥用。  
- **黑盒验证**：无需访问模型内部结构，可直接验证所有权。  
- **广泛实验验证**：覆盖72种配置、攻击场景与人类评估，充分证明方法稳定。

## 八、不足与局限

- **攻击极端场景**：若水印样本比例低于 20%，检测失效；对强混淆攻击（如 Tigress）仍能被清除。  
- **计算资源有限**：仅评估中型模型（≤1.3B参数），未验证更大模型的持续效果。  
- **语言覆盖限制**：仅测试 C / Java，虽理论可扩展但缺乏跨多语言实证。  
- **中毒可解释性风险**：水印去除引发性能下降可能被误判为自然偏差。  
- **伦理风险**：若水印机制公开，可能被恶意模仿或篡改水印以伪造所有权。
