## 摘要
多比特水印技术作为一种有前景的解决方案，可将不可察觉的二进制消息嵌入由大型语言模型（LLM）生成的文本中，从而实现对LLM的可靠归因与恶意使用的追踪。尽管近期取得了一定进展，现有方法仍存在关键局限：一些方法在处理大规模消息时计算代价过高，而另一些则在文本质量与解码准确率之间存在较差的平衡。此外，当生成文本中的token数量有限时，现有方法的解码准确率会显著下降，而这种情况在实际应用中频繁出现。为应对这些挑战，我们提出了\textsc{XMark}，一种在LLM生成的文本中编码与解码二进制消息的新方法。\textsc{XMark}编码器的独特设计在生成带水印的token时能产生较少失真的logit分布，从而保持文本质量，同时使其定制化解码器能够在token数量有限的情况下可靠地恢复编码消息。在多种下游任务上的大量实验表明，\textsc{XMark}在保持水印文本质量的同时显著提高了解码准确率，优于现有方法。代码地址：https://github.com/JiiahaoXU/XMark。

## Abstract
Multi-bit watermarking has emerged as a promising solution for embedding imperceptible binary messages into Large Language Model (LLM)-generated text, enabling reliable attribution and tracing of malicious usage of LLMs. Despite recent progress, existing methods still face key limitations: some become computationally infeasible for large messages, while others suffer from a poor trade-off between text quality and decoding accuracy. Moreover, the decoding accuracy of existing methods drops significantly when the number of tokens in the generated text is limited, a condition that frequently arises in practical usage. To address these challenges, we propose \textsc{XMark}, a novel method for encoding and decoding binary messages in LLM-generated texts. The unique design of \textsc{XMark}'s encoder produces a less distorted logit distribution for watermarked token generation, preserving text quality, and also enables its tailored decoder to reliably recover the encoded message with limited tokens. Extensive experiments across diverse downstream tasks show that \textsc{XMark} significantly improves decoding accuracy while preserving the quality of watermarked text, outperforming prior methods. The code is at https://github.com/JiiahaoXU/XMark.

- **研究动机**：随着大型语言模型（LLM）如 ChatGPT、LLaMA 被广泛应用，其生成高质量文本的能力也被恶意滥用（如虚假新闻、钓鱼邮件等）。

  - 零比特水印仅能检测 AI 文本，无法实现用户追踪与来源识别。

    1. 现有方法在消息较长时计算量过大。

    3. 当文本 token 较少时，解码准确度骤降，影响实际应用。
- **研究目标**：在保证文本自然性的同时提升水印解码准确率，尤其在短文本条件下仍保持可靠性。

## 二、方法论与技术设计

### 1. 核心思想

提出 **XMark** 方法，通过改进编码器与解码器，提升多比特黄水印的可解码性与文本质量。其关键创新包括：
- **“Leave-One-Shard-Out (LOSO)” 策略**：与传统方法仅标记一个分片为绿色不同，XMark将除目标分片外的全部分片设为“绿色”；
- **Evergreen List 设计**：通过多个词汇表排列的交集提高整体稳定性；
- **受约束的 Token–Shard 映射矩阵（cTMM）解码器**：通过限制每个token的计数次数提升解码精度。

### 2. 编码器流程概述

- 将待嵌入的消息 `m` 分为 `r` 个长度为 `d` 的块；
- 在每个 token 生成步骤：
  1. 获取模型 logits 分布；
  2. 使用 k 个不同的哈希键生成 k 个词汇表排列；
  3. 对每个排列分割为 2^d 个分片；
  4. 对应目标消息块的分片取补集形成 *绿色列表*；
  5. 交集形成 *Evergreen List*；
  6. 对位于 Evergreen List 中的 token logits 增加 bias δ；
  7. 从调整后的分布中采样生成水印 token。
- 理论结果：Evergreen List 的期望比例为 (1 - 2^-d)^k，可在保持较高文本质量的同时维持水印信号。

### 3. 解码器流程概述

- 初始化受约束矩阵 cTMM 为零；
- 对每个 token，根据前两 token 与哈希键重建分片结构；
- 每个 token 更新至对应 shard 的计数（每次最多增加 1）；
- 对每个消息块，选择计数最少的分片作为水印编码位置；
- 拼接各块得到完整的二进制消息。

### 4. 理论分析

- **提升质量**：由于绿色列表比例更大，生成的 logits 分布变形较少。
- **提升可靠性**：多哈希键与 evergreen 构造使得即使 token 较少，也能准确恢复原消息。
