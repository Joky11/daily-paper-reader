## 摘要
多比特水印技术已成为一种有前景的解决方案，可以将不可察觉的二进制信息嵌入大型语言模型（LLM）生成的文本中，从而实现对 LLM 的可靠溯源与恶意使用追踪。尽管近年来取得了一定进展，但现有方法仍存在关键局限：部分方法在信息量较大时计算不可行，另一些则在文本质量与解码准确率之间存在较差的平衡。此外，当生成文本的 token 数量有限时，现有方法的解码准确率会显著下降，而这种情况在实际应用中经常出现。为应对这些挑战，我们提出 \textsc{XMark}，一种用于在 LLM 生成文本中编码与解码二进制信息的新方法。\textsc{XMark} 的编码器独特设计能在生成带水印的 token 时产生较少失真的 logit 分布，从而保持文本质量，同时其定制化的解码器能在有限 token 条件下可靠恢复编码信息。我们在多个下游任务中的大量实验表明，\textsc{XMark} 在保持水印文本质量的同时显著提升了解码准确率，优于现有方法。代码可在 https://github.com/JiiahaoXU/XMark 获取。

## Abstract
Multi-bit watermarking has emerged as a promising solution for embedding imperceptible binary messages into Large Language Model (LLM)-generated text, enabling reliable attribution and tracing of malicious usage of LLMs. Despite recent progress, existing methods still face key limitations: some become computationally infeasible for large messages, while others suffer from a poor trade-off between text quality and decoding accuracy. Moreover, the decoding accuracy of existing methods drops significantly when the number of tokens in the generated text is limited, a condition that frequently arises in practical usage. To address these challenges, we propose \textsc{XMark}, a novel method for encoding and decoding binary messages in LLM-generated texts. The unique design of \textsc{XMark}'s encoder produces a less distorted logit distribution for watermarked token generation, preserving text quality, and also enables its tailored decoder to reliably recover the encoded message with limited tokens. Extensive experiments across diverse downstream tasks show that \textsc{XMark} significantly improves decoding accuracy while preserving the quality of watermarked text, outperforming prior methods. The code is at https://github.com/JiiahaoXU/XMark.

## 二、核心方法与技术细节

### 1. 总体框架
论文提出的方法命名为 **XMark**，其主要目标是实现：
- 可嵌入多比特二进制消息；
- 保持模型输出文本质量接近原始文本；
- 在有限 token 下仍能准确解码。

### 2. 编码器：Leave-One-Shard-Out (LOSO) 与 Evergreen List
- **传统问题（MPAC 的限制）**：MPAC 将一个 shard（词表分区）指定为“绿色列表”，并只提升其中 token 的采样概率，这导致绿名单过小→文本失真严重。
- **XMark创新**：使用 **Leave-One-Shard-Out (LOSO)** 策略——反向操作：保留除一个 shard 外的所有分区为绿色列表。这样绿色列表比例 γ 增大（≥0.75），显著降低了 logit 扭曲，提高了文本流畅度。
- **Evergreen List 设计**：通过多个不同 hash key（数量为 k）生成多次词表排列，然后将各绿色列表交集形成 evergreen list E。只有 E 内的 token 被加正偏置 δ。此交集操作使得信号稳健且干扰少。

  理论上，E 的期望比例：
  \[
  \mathbb{E}[\gamma] = (1 - 2^{-d})^k
  \]
  其中 d 为每块消息的比特数，k 为 hash key 数量。

### 3. 解码器：受限 Token–Shard 映射矩阵 (cTMM)
- 每个 suspect text 的 token 被映射回对应的多次词表分区；
- 解码器使用一个限制条件的矩阵（cTMM），保证每个 token 对 shard 的计数最多只增加一次；
- 对各消息块，选择 **token 计数最少的 shard** 作为未加偏置的 shard，从而恢复原始多比特消息；
- 与传统 TMM 相比，cTMM 有效抑制了计数爆炸问题，提升了有限 token 下的解码精度。

### 4. 算法流程
编码器伪流程如下（简述）：
1. 将消息 m 拆分为若干块 {m₀,…,mᵣ₋₁}；
2. 生成 k 个 hash key 并计算各自的种子；
3. 对每个种子进行词表扰动并分为 2ᵈ 个 shard；
4. 对所有 shard 执行 LOSO→产生绿色列表；
5. 取各绿色列表交集形成 evergreen list；
6. 对 E 中 token 的 logit 加偏置 δ；
7. 通过 softmax 采样生成 watermarked token。
