## 摘要
针对自回归（AR）图像生成的隐形水印技术近年来受到广泛关注，作为保护图像版权和追踪 AI 生成内容的一种手段。然而，现有方法存在三大局限性：（1）它们仅能嵌入用于二值验证的零比特水印，无法传递多比特信息；（2）它们依赖静态的代码簿划分策略，一旦划分方式被公开，便容易受到安全攻击；（3）它们通常针对特定的 AR 架构设计，缺乏在不同 AR 范式间的泛化能力。我们提出了 \method{}，一种无需训练的、面向自回归图像生成器的统一水印框架，能够同时解决上述三项问题。\method{} 引入了三个核心组件：\textbf{自适应语义分组（Adaptive Semantic Grouping, ASG)}，依据语义相似度和密钥动态划分代码簿条目，以确保图像质量与安全性；\textbf{分块多比特编码（Block-wise Multi-bit Encoding, BME)}，将 token 序列划分为多个块，并通过纠错编码在不同块间传递比特信息，实现可靠的消息传输；以及\textbf{统一 token 替换接口（Unified Token-Replacement Interface, UTRI)}，抽象化水印嵌入过程，使其同时支持下一 token 预测（如 LlamaGen）和下一尺度预测（如 VAR）两种范式。我们对检测误差率和嵌入容量进行了理论分析。大量实验结果基于三种 AR 模型表明，\method{} 在图像质量（FID）、水印检测精度以及多比特消息提取方面均达到了最新的性能水平，同时对裁剪、JPEG 压缩、高斯噪声、模糊、颜色抖动和随机擦除等攻击表现出较强的鲁棒性。

## Abstract
Invisible watermarking for autoregressive (AR) image generation has recently gained attention as a means of protecting image ownership and tracing AI-generated content. However, existing approaches suffer from three key limitations: (1) they embed only zero-bit watermarks for binary verification, lacking the ability to convey multi-bit messages; (2) they rely on static codebook partitioning strategies that are vulnerable to security attacks once the partition is exposed; and (3) they are designed for specific AR architectures, failing to generalize across diverse AR paradigms. We propose \method{}, a training-free, unified watermarking framework for autoregressive image generators that addresses all three limitations. \method{} introduces three core components: \textbf{Adaptive Semantic Grouping (ASG)}, which dynamically partitions codebook entries based on semantic similarity and a secret key, ensuring both image quality preservation and security; \textbf{Block-wise Multi-bit Encoding (BME)}, which divides the token sequence into blocks and encodes different bits across blocks with error-correcting codes for reliable message transmission; and \textbf{a Unified Token-Replacement Interface (UTRI)} that abstracts the watermark embedding process to support both next-token prediction (e.g., LlamaGen) and next-scale prediction (e.g., VAR) paradigms. We provide theoretical analysis on detection error rates and embedding capacity. Extensive experiments on three AR models demonstrate that \method{} achieves state-of-the-art performance in image quality (FID), watermark detection accuracy, and multi-bit message extraction, while maintaining robustness against cropping, JPEG compression, Gaussian noise, blur, color jitter, and random erasing attacks.

## 2. 方法论：UniMark 框架

### 2.1 整体思路
UniMark（Unified Adaptive Multi-bit Watermarking）通过在 AR 图像生成的 token 级过程中引入**动态分组与多比特编码机制**，实现在不牺牲图像质量的前提下嵌入可提取的水印比特流。  
该方法完全无需额外训练，可直接作用于任意 AR 模型。

### 2.2 三个核心组件

#### （1）自适应语义分组（Adaptive Semantic Grouping, ASG）
- **思想**：利用 codebook 中各 token 的语义嵌入相似性，动态地将其分组为“绿色集(G)”与“红色集(R)”。
- **步骤**：
  - 计算所有 codebook 向量之间的 **余弦相似度矩阵** \( s_{ij} = \cos(e_i, e_j) \)。
  - 基于**密钥 κ 和位置 i** 生成伪随机种子 \( seed_i = H(κ∥i) \)，并据此随机打乱索引形成不同分区。
  - 对每个 token 位置 i，设置绿色比例参数 γ，得到：
    \[
    G_i = \text{前} \, \lfloor \gamma K \rfloor \, \text{个token},\quad R_i = C \setminus G_i
    \]
  - 若当前生成 token 不在 G_i 中，则替换为语义最相似的绿色 token。
- **特性**：  
  每一位置使用独立分区 → 提高安全性；语义相似替代 → 保持图像质量。

#### （2）分块多比特编码（Block-wise Multi-bit Encoding, BME）
- **思想**：通过将 token 序列划分为多块，每块携带一个比特信息，并使用 **BCH 纠错码** 保障可靠提取。
- **流程**：
  1. 输入消息 m（长度 B 位），用 BCH(n, B, d) 编码为 m′；  
  2. 将全部 N 个 token 均分为 n 个块 B_j；  
  3. 对每块：
     - 若编码位 \( m'_j=1 \)，偏向 G 集；
     - 若 \( m'_j=0 \)，偏向 R 集；
  4. 提取时重新编码图像为 token 序列，统计块内 token 属于 G 集的比例 \(r_j\)，并以阈值 τ=0.5 判定比特。
- **扩展**：零比特检测即统计整个序列的总体绿色比例 z，并基于正态分布检验判定。

#### （3）统一 token 替换接口（Unified Token-Replacement Interface, UTRI）
- **目的**：统一不同 AR 范式。  
  - 对 **next-token 模型（如 LlamaGen）**：直接替换一维序列中的 token。  
  - 对 **next-scale 模型（如 VAR）**：整合多尺度 token 图，将粗尺度到细尺度依次分配到块中。  
- **作用**：使算法可无缝兼容多种自回归架构。

### 2.3 理论分析
- **错误率分析**：在未加水印的图像中，token 出现在 G 集的概率为 γ，按中心极限定理可推得误检率 α。  
- **容量分析**：给出可可靠嵌入的最大比特数：
  \[
  B_{max} = \Big\lfloor \frac{N}{N_{min}} \Big\rfloor · R_{BCH}
  \]
  其中 \(N_{min}\) 为统计置信要求的最小块长度，体现图像 token 数 N 越大则容量越高。

### 2.4 算法流程
**算法1（嵌入）**：  
获取 token 序列 → BCH 编码 → 分块 → 基于密钥生成分组并替换 → 解码生成图像。  
**算法2（提取）**：  
重新编码图像 → 计算绿色比例 → 阈值判定 → BCH 纠错恢复信息。
