## 摘要
代码类大语言模型（Code LLMs）已经彻底革新了软件开发方式，但同时引发了关于代码来源、版权保护和安全性的重大担忧。现有的代码水印方法存在两个根本性局限：黑盒方法要么表现出易于统计分析检测的语法模式，要么依赖神经网络隐式嵌入行为，削弱了解释性、可审计性和精确控制；而白盒方法则缺乏代码感知能力，可能影响代码功能。此外，当前的单层水印方案无法应对日益复杂的来源追溯需求，如多层级归属和版本追踪。我们提出了 MATRIX，这是一种新颖的代码水印框架，将水印编码形式化为求解受约束的校验矩阵方程。MATRIX 通过变量命名与语义保持转换实现双通道水印，从而在更广泛的代码范围内增强水印覆盖，同时提供互为备份的鲁棒性。通过将 BCH 纠错码与解空间多样性相结合，我们的方法在抵抗统计分析方面具备卓越的稳健性。在多种 Code LLM 生成的 Python 代码上的广泛评测表明，MATRIX 在功能损失最小（0–0.14%）的情况下，实现了平均 99.20% 的水印检测准确率，针对多种攻击的鲁棒性提升了 7.70–26.67%，水印适用范围相较现有方法提升了 2–6 倍。结果表明，MATRIX 在平衡可检测性、保真度和鲁棒性的同时，为复杂代码溯源场景提供了一种高效的解决方案。

## Abstract
Code Large Language Models (Code LLMs) have revolutionized software development but raised critical concerns regarding code provenance, copyright protection, and security. Existing code watermarking approaches suffer from two fundamental limitations: black-box methods either exhibit detectable syntactic patterns vulnerable to statistical analysis or rely on implicit neural embedding behaviors that weaken interpretability, auditability, and precise control, while white-box methods lack code-aware capabilities that may compromise functionality. Moreover, current single-layer watermarking schemes fail to address increasingly complex provenance requirements such as multi-level attribution and version tracking.   We present MATRIX, a novel code watermarking framework that formulates watermark encoding as solving constrained parity-check matrix equations. MATRIX employs dual-channel watermarking through variable naming and semantic-preserving transformations, enhancing watermark coverage across a wider range of code while ensuring mutual backup for robustness. By integrating BCH error-correction codes with solution space diversity, our approach achieves robustness against statistical analysis. Extensive evaluation on Python code generated by multiple Code LLMs demonstrates that MATRIX achieves an average watermark detection accuracy of 99.20% with minimal functionality loss (0-0.14%), improves robustness by 7.70-26.67% against various attacks, and increases watermarking applicability by 2-6x compared with existing methods. These results establish MATRIX as an effective solution for complex code provenance scenarios while balancing among detectability, fidelity, and robustness.

## 一、研究背景与核心问题

- **研究背景**  
  代码类大语言模型（Code LLMs）已在自动代码生成、补全、调试等任务中广泛应用，推动软件开发方式的革新。然而，其生成的代码可能涉及：
  - 版权和原创性问题（可能无意中复现训练集中的受版权保护代码）；  
  - 安全漏洞风险（生成的代码可能存在隐患被恶意利用）；  
  - 溯源与责任归属困难。  

- **核心问题**  
  现有的代码水印技术存在显著局限：  
  1. **黑盒方法**：依赖固定语法转换或模型隐式分布，容易被统计分析检测；解释性差、不可控。  
  2. **白盒方法**：在模型解码过程中插入“绿色/红色列表”，缺乏代码语义分析能力，破坏功能。  
  3. **单层水印**：无法支持复杂的多层归属（如组织和个人双重标识）与版本追踪场景。  

  因此，论文的目标是：  
  > 构建一种**可解释、可控、鲁棒、支持多层溯源的代码水印框架**，在保持代码语义与功能正确的同时，实现高检测率与隐蔽性。

## 二、方法论与技术路线

### 1. 基本思想
- 作者提出 **MATRIX** 框架，将代码水印嵌入问题形式化为“**受约束的校验矩阵方程求解**”（Constrained Parity-Check Matrix Solving）。
- 通过**双通道设计**提升鲁棒性与覆盖率：
  - **自然通道（Natural Channel）**：基于变量命名变换（如大小写、下划线、后缀）。  
  - **形式通道（Formal Channel）**：基于语义保持的代码转换（如循环结构改写、列表推导式替换）。  
  - 两通道可相互备份，提高在攻击下的恢复能力。

### 2. 编码机制
- **水印编码步骤**：
  1. 将身份信息 \( m \in \{0,1\}^k \) 通过 **BCH 编码**转化为更长的冗余序列 \( w = BCHEncode(m) \)，确保纠错能力与隐蔽性。
  2. 定义一系列语义保持**转换规则集 \( T \)**。每个规则对应一个**锚点(anchor point)**。
  3. 确定每个锚点是否应用转换（状态向量 \( r=(s_1,\dots,s_n) \)），并依据：
     \[
     M \cdot r \equiv c \ (\text{mod } 2)
     \]
     其中 \( M \) 是约束校验矩阵，\( c \) 是分组状态向量，与编码序列 \( w \) 对应。

- **嵌入与提取算法流程（简述）**
  - *嵌入时*：
    1. 识别可转换锚点；
    2. 依据编码容量选择合适的 BCH 序列与校验矩阵；
    3. 求解方程得到状态向量；
    4. 应用前向或反向转换生成水印代码。  
  - *提取时*：
    1. 重新识别锚点并还原状态；
    2. 校验是否满足校验方程；
    3. 若满足则判定为含水印代码，重构身份信息。

- **技术创新**  
  - “解空间多样性”设计使得同一水印可有多种合法编码方式，有效抵抗统计模式推断。  
  - 双通道冗余避免单点失效，提高鲁棒性。  
  - 仅需更改校验矩阵即可支持多组织层级溯源，无需重写系统。

## 八、不足与局限

- **语法完整性依赖**：静态分析要求代码为可解析单元，难以处理零散或不完整片段。  
- **手工规则依赖**：语义保持转换集需人工定义，覆盖面有限。  
- **锚点数量限制**：当可用锚点极少时，多层追踪能力下降。  
- **高复杂场景未验证**：论文主要基于 Python，未测其他语言的适应性。  
- **高级对手模型风险**：在极端情况下（攻击者知晓部分规则）仍可能设计规避策略。
