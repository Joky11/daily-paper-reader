## 摘要
随着大型语言模型逐渐成为内容生成的标准后端，实际的内容溯源需求日益要求实现多比特水印。在提供商内部部署场景中，一个关键要求是消息对称性：嵌入的消息本身不应系统性地影响文本质量或验证结果。基于词汇划分的水印方法在低熵解码下可能破坏消息对称性：某些消息被分配到大部分概率质量，而其他消息被迫使用尾部词元。这使得嵌入质量和消息解码准确率依赖于特定的消息。我们提出了 QuantileMark，一种在连续累积分布区间 $[0, 1)$ 中嵌入消息的白盒多比特水印。在每一步中，QuantileMark 将该区间划分为 $M$ 个等概率区间，并严格从分配给目标符号的区间中进行采样，从而确保无论上下文熵如何变化，均拥有固定的 $1/M$ 概率预算。对于检测，验证方在教师强制（teacher forcing）模式下重建相同划分，计算潜在区间的后验概率，并聚合证据以进行验证。我们证明了消息无偏性（message-unbiasedness）这一性质，保证对消息进行平均时可恢复基础分布。这为生成端对称性提供了理论基础，而等概率设计进一步在检测端促进了消息间证据强度的均匀性。在 C4 continuation 和 LFQA 上的实验证明，QuantileMark 在多比特恢复与检测鲁棒性方面优于强基线方法，对生成质量的影响可忽略不计。我们的代码可在 GitHub (https://github.com/zzzjunlin/QuantileMark) 获取。

## Abstract
As large language models become standard backends for content generation, practical provenance increasingly requires multi-bit watermarking. In provider-internal deployments, a key requirement is message symmetry: the message itself should not systematically affect either text quality or verification outcomes. Vocabulary-partition watermarks can break message symmetry in low-entropy decoding: some messages are assigned most of the probability mass, while others are forced to use tail tokens. This makes embedding quality and message decoding accuracy message-dependent. We propose QuantileMark, a white-box multi-bit watermark that embeds messages within the continuous cumulative probability interval $[0, 1)$. At each step, QuantileMark partitions this interval into $M$ equal-mass bins and samples strictly from the bin assigned to the target symbol, ensuring a fixed $1/M$ probability budget regardless of context entropy. For detection, the verifier reconstructs the same partition under teacher forcing, computes posteriors over latent bins, and aggregates evidence for verification. We prove message-unbiasedness, a property ensuring that the base distribution is recovered when averaging over messages. This provides a theoretical foundation for generation-side symmetry, while the equal-mass design additionally promotes uniform evidence strength across messages on the detection side. Empirical results on C4 continuation and LFQA show improved multi-bit recovery and detection robustness over strong baselines, with negligible impact on generation quality. Our code is available at GitHub (https://github.com/zzzjunlin/QuantileMark).

## 一、研究动机与问题背景

- **核心问题**：  
  随着大型语言模型（LLM）广泛用于内容生成，如何确保生成内容的可追溯性（provenance）成为关键问题。传统的水印技术多为“零比特”方案，仅判断是否存在水印，但实际应用需要能编码多比特信息（如用户 ID、模型版本、时间戳等）。

- **主要挑战**：  
  在多比特水印中，必须保证不同消息（即不同嵌入比特组合）不会导致生成文本在质量或验证上的系统性差异，这被称为**消息对称性（message symmetry）**。然而，常见的词汇划分（vocabulary partition）方法在“低熵解码”场景下会造成严重不对称，一些消息会被分配到高概率词元，而另一些被迫采用低概率尾部词元，造成文本质量下降和验证不稳定。

- **研究目标**：  
  创建一种能在白盒环境下实现多比特嵌入，同时保持消息对称性与检测鲁棒性的水印方案。

## 二、方法论：QuantileMark 的核心思想与技术机制

### 2.1 核心思想

- QuantileMark提出一种**基于连续概率累积区间的水印嵌入机制**，用“等质量分区（equal-mass quantile partition）”替代传统的词汇划分策略。
- 每一步生成过程中，将累计概率区间 $[0,1)$ 划分为 $M=2^m$ 个等概率区间（即每个区间含相同概率质量）。
- 目标消息符号（symbol）对应唯一的区间编号，采样随机数时仅在该区间内取样，从而保证每条消息在任何上下文下都享有相同的概率预算 $1/M$。

### 2.2 技术流程（简述）

1. **消息与位置分配**：  
   使用密钥 \(K\) 产生消息符号序列 \((s_1, ..., s_H)\)；每个生成步 \(t\) 通过伪随机函数将当前上下文映射至嵌入位置索引 \(i_t\)。

2. **嵌入阶段（Embedding）**：
   - 计算当前上下文的 token 概率分布 \(p_0(\cdot | h_t)\)，得到相应 CDF。
   - 将 CDF 对应的区间映射为等质量 bin：\(B_r = [r/M, (r+1)/M)\)。
   - 目标符号通过密钥映射到目标区间 \(r_t^\star = \varphi_t(s_{i_t})\)。
   - 在该区间内采样随机数 \(u_t \sim \text{Unif}(B_{r_t^\star})\)，选择覆盖此位置的 token 作为输出，即实现嵌入。

3. **检测阶段（Detection）**：
   - 在 teacher forcing 模式下重建模型概率分布 \(p_0(\cdot | h_t)\)，再计算各区间的 **重叠概率质量** \(\mu_t(x_t, r)\)。
   - 得到每个 token 的后验：  
     \(p_t(r) = \mu_t(x_t, r) / p_0(x_t | h_t)\)。
   - 各步证据通过 **对数后验比（Log Posterior Odds, LPO）** 聚合：
     \[
     \mathrm{LPO}_t(r) = \log\frac{p_t(r)}{1 - p_t(r)}
     \]
   - 最终在所有步上累积并解码最可能的消息 \(\hat{S}\)，计算整体检测得分 \(T(x_{1:T})\)。

### 2.3 理论性质

- **消息无偏性（Message-unbiasedness）**：  
  在符号均匀分布条件下，平均后等分区方案可恢复原始分布 \(p_0(v|h_t)\)，确保生成侧分布不受消息影响。

- **密钥无偏性（Cipher-unbiasedness）**：  
  当密钥引入的随机重映射满足均匀分布时，固定消息下平均后仍可恢复原始分布。

- **理论优点**：  
  提供消息与密钥层面双重无偏性证明，保证“生成质量与消息独立”。

## 八、不足与局限

- **鲁棒性限制**：  
  在强重写或大量 paraphrase 攻击下性能仍明显下降，原因在于上下文漂移破坏位置分配与几何重建。

  当前方法假定“白盒访问”，即提供方在验证时需模型 logits，暂不适用于仅文本可得的黑盒检测。
- **覆盖范围有限**：  
  实验主要集中于两个英文数据集及 7B/8B 级模型，尚未验证跨语言、多任务或更大模型的适应性。

  论文未公开资源消耗与硬件配置，难以评估大规模部署成本。
- **消息扩展性**：  
  当消息长度（比特数）过大时，每符号支持 token 减少，导致信息稀释现象，需后续研究纠错方案。

  虽设计用于提供方内部追溯，但嵌入用户级 ID 的机制仍需严格政策监管。
