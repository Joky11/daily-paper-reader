## 摘要
多比特水印技术已成为在大型语言模型（Large Language Model, LLM）生成文本中嵌入不可察觉的二进制信息的一种有前景的解决方案，从而实现对LLM的可靠归因以及对其恶意使用的追踪。尽管近期取得了一些进展，现有方法仍面临关键限制：部分方法在处理较长消息时计算量过大，而另一些方法则在文本质量与解码准确率之间存在不佳的权衡。此外，当生成文本的token数量受限时（这一情况在实际应用中十分常见），现有方法的解码准确率会显著下降。为应对这些挑战，我们提出了\textsc{XMark}，一种用于在LLM生成文本中编码和解码二进制信息的新方法。\textsc{XMark}的编码器独特设计可以在生成带水印的token时产生较少失真的logit分布，从而保持文本质量，同时使其定制化的解码器能够在有限token条件下可靠地恢复嵌入的信息。我们在多种下游任务上的广泛实验表明，\textsc{XMark}在保持水印文本质量的同时，显著提升了解码准确率，优于现有方法。代码可在 https://github.com/JiiahaoXU/XMark 获取。

## Abstract
Multi-bit watermarking has emerged as a promising solution for embedding imperceptible binary messages into Large Language Model (LLM)-generated text, enabling reliable attribution and tracing of malicious usage of LLMs. Despite recent progress, existing methods still face key limitations: some become computationally infeasible for large messages, while others suffer from a poor trade-off between text quality and decoding accuracy. Moreover, the decoding accuracy of existing methods drops significantly when the number of tokens in the generated text is limited, a condition that frequently arises in practical usage. To address these challenges, we propose \textsc{XMark}, a novel method for encoding and decoding binary messages in LLM-generated texts. The unique design of \textsc{XMark}'s encoder produces a less distorted logit distribution for watermarked token generation, preserving text quality, and also enables its tailored decoder to reliably recover the encoded message with limited tokens. Extensive experiments across diverse downstream tasks show that \textsc{XMark} significantly improves decoding accuracy while preserving the quality of watermarked text, outperforming prior methods. The code is at https://github.com/JiiahaoXU/XMark.

## 二、方法论与技术细节

### 1. 核心思想

提出新颖的多比特水印框架 **XMark**，在LLM生成过程中动态修改部分token logits，实现**高质量文本生成与高可靠解码**的兼容。  
核心创新点包括三个步骤：

1. **Leave-One-Shard-Out（LOSO）编码策略**：  
   - 将词表划分为若干“碎片”（shards）。  
   - 不同于传统方法仅提升目标碎片概率（green list），LOSO反其道而行——**提升除目标碎片外的所有token概率**，保留一个未加权的shard作为隐含水印标志。  
   - 好处：显著扩大“绿色列表比例”（γ = 1 - 2⁻ᵈ ≥ 0.75），减少对采样分布的扭曲，提高文本质量。

2. **多重哈希+Evergreen List融合机制**：  
   - 使用多个独立哈希密钥（k个）生成不同词表排列，并执行LOSO操作；  
   - 将多个green lists取交集形成**evergreen list**，在采样阶段仅对其中的tokens施加偏置δ。  
   - 理论期望：Evergreen list的规模约为(1 - 2⁻ᵈ)ᵏ|V|，能在增加解码观察数的同时保持分布稳定。

3. **受约束Token-Shard映射矩阵（cTMM）解码器**：  
   - 在解码时记录每个token出现于不同shard的计数，但限制每条计数最多增加一次，防止过度映射导致混淆。  
   - 最终通过统计各块中**最少出现**的shard来还原二进制信息，实现鲁棒解码。

### 2. 算法流程概述

- **编码过程**：
  1. 将消息 m 分割为 r 个 d位的块；
  2. 计算多哈希种子并生成k个独立词表排列；
  3. 每个排列执行LOSO策略形成green lists；
  4. 求交集得到evergreen list；
  5. 对evergreen list中token logits加δ偏置；
  6. 采样下一token。

- **解码过程**：
  1. 使用相同的哈希函数重建词表分区；
  2. 累积每个token的shard计数形成cTMM；
  3. 从计数最少的shard推断出嵌入位；
  4. 拼接成完整二进制消息。
